================================================================================
ENDPOINT MEDIA — CONTENT CONSOLE (csmm / csmmendpoint)
================================================================================

WHAT THIS PROJECT IS
--------------------------------------------------------------------------------
A Streamlit-based internal tool for a social/content studio ("Endpoint Media").
It stores client brand profiles in SQLite, runs AI-assisted post generation
(CrewAI + Gemini) per client, and tracks manual distribution workflow: captions,
1:1 and 9:16 image prompts, optional overlay JSON, final exported images, QC,
and a separate "publisher" view for someone who posts to Facebook without
needing Meta page access.

Core idea: one place to generate and approve content, hand off assets + copy
to a publisher via a private link (query param + shared key), not auto-post
to Facebook in Phase 1.

KEY FILES
--------------------------------------------------------------------------------
  src/app.py          Main UI, theme CSS, navigation, generation hub, post library,
                      publisher queue view (?view=publisher&key=...)
  src/main.py         Entry: local runs spawn Streamlit on app.py; Cloud runs
                      app in-process (no nested Streamlit server).
  src/database.py     SQLite schema, migrations, posts/clients CRUD, QC/publisher
                      fields, content gap analysis helpers.
  src/seed_clients.py Default clients inserted when DB has zero clients (fresh deploy).
  src/crew.py         CrewAI social media crew wiring.
  src/config/         Brand vault (featured manufacturers), vertical creative rules
                      (tyre vs battery vs firewood clients).
  static/titan_ad_exporter.html   "Overlay studio" — browser HTML exporter (html2canvas).
  .streamlit/config.toml           Streamlit theme tokens (aligns with app CSS).
  agency.db           Local SQLite file (gitignored). Cloud gets its own ephemeral DB.
  .env.example        Documents env vars; real secrets go in .env (local) or
                      Streamlit Cloud Secrets (deployed).

================================================================================
USER-FACING VIEWS (SIDEBAR NAVIGATION)
================================================================================

The left sidebar is the primary navigation. Structure:

  Endpoint Media          (title — Crimson Pro, large)
  Content studio          (caption — muted)

  [ Dashboard ]           Primary nav — generation + post library for selected client
  [ New client ]          Onboarding form — add/edit client DNA (no generation chrome)
  [ Overlay studio ]      Full-width embed of Titan-style HTML ad exporter

  CLIENTS                 (section heading — uppercase, letter-spaced, muted)
  [ Client name ]         One button per client; active client highlighted (navy).
                          Clicking a client switches active client and returns to Dashboard.

Session state:
  - current_view: Dashboard | New client | Overlay studio
  - em_active_client_id: which client is selected for Dashboard generation/library

SPECIAL URL (NO SIDEBAR FLOW)
--------------------------------------------------------------------------------
  ?view=publisher&key=<PUBLISHER_SHARED_KEY>
  External publisher queue: posts marked QC "Ready for Publisher" for clients
  matching PUBLISHER_QUEUE_CLIENTS. Download ZIP, copy caption, track schedule/posted.

================================================================================
DASHBOARD LAYOUT (MAIN WORK AREA) — "Dashboard" VIEW
================================================================================

When "Dashboard" is selected and at least one client exists:

1) HERO (top)
   - Title: "Endpoint Media — Content console" (class ui-hero)
   - Subtitle explaining: pick client in sidebar, generate posts, copy assets,
     track delivery (class ui-hero-sub)

2) ACTIVE CLIENT STRIP
   - Line: "Active client: <name> · profile #<id>" (class ui-active-client)
   - Left accent bar in bronze (C_ACCENT); border and light surface background

3) CONTENT GAPS (last 30 days)
   - Heading + gap analysis alerts from database (pillars / featured brands vs recent posts)
   - Each alert: info/warning styling; optional "Generate Post to Fill Gap" button

4) GENERATION HUB (contained "card" region)
   - Wrapped in a bordered, shadowed container (em_generation_hub styling)
   - Typically includes:
     - Post format: Standard Feed (1:1) vs Story (9:16)
     - Content pillar, featured brand (vault), creative hook (options vary by vertical:
       automotive tyre vs battery vs firewood)
     - Batch count, mixed-format options, battery product line (battery vertical only)
     - Plan summary panel (em-panel): explains what will run for this client
     - Buttons: "Run generation" (primary navy), "Auto-Pilot (random settings)"

5) POST LIBRARY (below generation)
   - Section header styled as audit block (em-audit-header-*):
     "Post library" / subtitle about asset delivery & tracking
   - Summary table (Streamlit dataframe): post #, created, pillar, brand, format,
     delivery workflow status, QC status
   - Per-post cards (em-card):
     - Suggested caption (read-only text area) + Copy button (clipboard iframe)
     - 1:1 and 9:16 AI image prompts + Copy buttons
     - Optional expander: suggested text overlay JSON
     - Meta line: pillar, brand, format
     - Final images: upload square + vertical, Save, Mark ready for publisher,
       Revert QC to draft
     - Delivery status select: Draft / Sent to Client / Posted

VISUAL DESIGN SYSTEM — COLORS (DESIGN TOKENS IN src/app.py)
================================================================================

Philosophy: "Classical editorial" — warm paper background, ink-like text,
restrained navy primary and bronze accent (not neon startup UI).

Token name              Hex         Role
--------------------------------------------------------------------------------
C_BG                    #f5f0e8     App page background (warm paper)
C_SURFACE               #fffcf7     Cards, inputs, sidebar lift, alerts
C_TEXT                  #2c2416     Primary ink (headings emphasis)
C_TEXT_BODY             #4a433a     Body copy
C_MUTED                 #6b6459     Captions, labels, secondary text
C_BORDER                #cdc6b9     Borders, dividers
C_INPUT_BG              #fffdf8     Input fields
C_PRIMARY               #1e3a5f     Navy — primary buttons, H2/H3, sidebar active
C_ACCENT                #7c5c36     Bronze — left accent bar on active client strip
C_HEADING               #1c1917     Hero / page titles
C_SUCCESS               #2f5f46     Success-adjacent (legacy name)
C_SHARK                 #ebe6dc     Panel background (plan summary panels)

Streamlit native theme (.streamlit/config.toml) mirrors these:
  primaryColor #1e3a5f, backgroundColor #f5f0e8, secondaryBackgroundColor #fffcf7,
  textColor #2c2416, base light, toolbar minimal.

TYPOGRAPHY
================================================================================

Fonts loaded from Google Fonts (injected in _inject_classical_theme):
  - Crimson Pro (serif): page titles, sidebar title, section headers, audit titles
  - Source Sans 3 (sans): body, labels, buttons, inputs, tables

Hierarchy:
  - Main h1: ~2rem, Crimson Pro semibold, C_HEADING
  - h2/h3/h4: Crimson Pro, C_PRIMARY
  - Body: 17px default, C_TEXT_BODY
  - Captions: ~0.92rem, C_MUTED
  - Sidebar section "Clients": uppercase micro-label, letter-spacing, C_MUTED

================================================================================
DATA & WORKFLOW (BEHAVIORAL MODEL)
================================================================================

Clients: company profile, industry, brand_context, tone, services_list,
         target_markets, photography_style

Posts:  caption, combined image_prompt (legacy), separate 1:1 and 9:16 prompts,
        suggested_text_overlay JSON, content pillar, featured brand, post format,
        workflow_status (Draft / Sent to Client / Posted),
        final image paths (square + vertical), qc_status (Draft / Ready for Publisher),
        publisher_status, scheduled_for, notes, timestamps.

Seeding: If the clients table is empty on init_db(), seed_clients.py adds
         Alberton Tyre Clinic, Alberton Battery Mart, Miwesu Fire wood (full briefs).

Publisher: PUBLISHER_SHARED_KEY + PUBLISHER_QUEUE_CLIENTS (comma-separated name
           fragments) gate the external view.
